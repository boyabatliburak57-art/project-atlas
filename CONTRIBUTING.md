# Katkı Kuralları

## Branch'ler

- `main`: kararlı sürüm
- `develop`: entegrasyon
- `feature/<ad>`
- `fix/<ad>`
- `docs/<ad>`
- `chore/<ad>`

## Commit mesajları

Conventional Commits kullanılır: `feat`, `fix`, `docs`, `refactor`, `test`, `perf`, `build`, `ci`, `chore`.

## Pull request

Amaç, etkilenen modüller, test sonucu, doküman etkisi ve kırıcı değişiklik bilgisi yazılmalıdır.

## Yasaklar

- secret değerleri repoya eklemek
- dokümansız veri modeli değişikliği
- test olmadan kritik finansal hesap eklemek
- lisansı belirsiz kod veya veri kullanmak
